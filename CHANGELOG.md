# Changelog

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/),
проект следует [семантическому версионированию](https://semver.org/lang/ru/).

## [Unreleased]

### Added
- **Визуальная вкладка «Маршрут»** в обоих UI: строка-вердикт «где начинаются
  потери» (узел + зона) и цветной бар потерь у каждого хопа — проблемный участок
  виден с одного взгляда. На Android вкладка переписана со списка строк на
  карту хопов; в TUI добавлены баннер и колонка-гейдж потерь.
- `probe.Diagnosis.FirstIssueHop` / `FirstIssueLoss` — структурный «виновный хоп»
  (TTL + потери/задержка), общий источник для баннера обоих фронтендов.

## [0.2.0] — 2026-06-17

### Added
- **Android-приложение** — GUI на [Fyne](https://fyne.io) поверх того же
  `internal/probe`, без переписывания ядра: те же четыре вкладки
  (Пинг / Маршрут / Диагноз / Скорость). APK собирается имеющимися SDK+NDK через
  `make apk` (без Android Studio), десктоп-отладка — `make gui`. ICMP на Android
  работает тем же unprivileged-сокетом (проверено на устройстве).
- Общий классификатор качества `probe.Quality`: пороги вердикта (потери/джиттер)
  вынесены в `internal/probe`, так что TUI и мобильный UI больше не расходятся.

## [0.1.0] — 2026-06-17

Первый релиз. Полный набор возможностей:

### Added
- TUI на Bubble Tea, **четыре вкладки**: «Пинг», «Маршрут» (mtr-стиль),
  «Диагноз», «Скорость».
- **Пинг-монитор**: RTT, потери, джиттер (RFC 3550), спарклайн истории и вердикт
  качества в 4 уровня (Отлично / Хорошо / Плохо / Критично) с причиной; считается
  по скользящему окну последних проб.
- **Маршрут**: per-hop потери/задержки/СКО, параллельный пул проберов;
  авто-подсветка `⚠` persistent-аномалий с приростом задержки `+ΔX`; AS-имя сети
  у каждого хопа (Team Cymru DNS) и фоновый reverse-DNS.
- **Диагноз**: маршрут сегментируется по зонам (локальная сеть → провайдер →
  транзит → назначение) с вердиктом по каждой и строкой «маршрут здоров» /
  «проблема в зоне «X»».
- **Скорость**: латентность, download ↓ и upload ↑ через открытые endpoint'ы
  Cloudflare.
- Чистый Go ICMP за интерфейсом `prober`: unprivileged datagram-сокет на
  macOS/Linux (без root), `iphlpapi.IcmpSendEcho` на Windows (без админки).
- **One-shot режим** `--once`: отчёт в stdout текстом (для тикетов ISP) или
  `--json` (стабильный wire-формат для cron); опции `--duration`, `--no-speed`.
  Реализация — `internal/report/`.
- `Makefile` с кросс-сборкой (darwin/linux/windows, amd64/arm64/386), стрипом и
  инъекцией версии; флаг `-version`.
- Документация: README, [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md),
  [docs/ROADMAP.md](docs/ROADMAP.md), CONTRIBUTING.
- Тесты: рендер вкладок, юнит-логика (аномалии, диагноз, ASN-кэш с фейк-часами,
  скользящее окно, wire-формат), живой сетевой тест (gated), детектор гонок в CI.

### Fixed
- Вердикт качества считается по скользящему окну, а не накопительно за сессию —
  одна ранняя потеря больше не «залипает» на минуты.
- ASN: неудачный резолв ретраится по TTL (60 с) вместо кэширования навсегда.
- `--once`: ошибки probe-слоя теперь видны в отчёте (раньше секция молча
  пропускалась — выглядело как «всё ок»); добавлена валидация `-duration > 0`.
- Windows: реальная ошибка ядра ICMP отличается от таймаута и пробивается в `Err`.
- Устранены утечки горутин/записей в фоновых кэшах (`dnsCache`/`asnCache`) и
  гонка данных в speed-тесте; фоновые резолвы прерываются по отмене контекста.

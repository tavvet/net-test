# Changelog

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/),
проект следует [семантическому версионированию](https://semver.org/lang/ru/).

## [Unreleased]

### Added
- TUI на Bubble Tea с тремя вкладками: «Пинг», «Маршрут» (mtr-стиль), «Скорость».
- Пинг-монитор: RTT, потери, джиттер (RFC 3550), история-спарклайн, вердикт.
- Трассировка маршрута: per-hop потери/задержки/СКО, параллельный пул проберов,
  фоновый reverse-DNS.
- Тест скорости через Cloudflare: латентность (с прогревом), download и upload.
- Чистый Go ICMP за интерфейсом `prober`:
  - unprivileged datagram-сокет на macOS/Linux (без root);
  - бэкенд на `iphlpapi.IcmpSendEcho` для Windows (без админки).
- `Makefile` с кросс-сборкой (darwin/linux/windows, amd64/arm64/386), стрипом и
  инъекцией версии; флаг `-version`.
- Документация: README, [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md), CONTRIBUTING.
- Тесты: рендер вкладок, живой сетевой тест (gated), детектор гонок.

### Changed
- Вердикт качества (вкладка «Пинг»): четыре уровня
  (Отлично / Хорошо / Плохо / Критично) с указанием причины, например
  `Качество: Плохо (потери 3.2%)`. Severity = максимум по факторам (потери, джиттер);
  абсолютный RTT намеренно не учитывается.
- Подсветка аномалий на вкладке «Маршрут»: проблемный хоп помечается `⚠`,
  колонка «Сред» получает суффикс `+ΔX ms` с приростом задержки. Используется
  persistent-эвристика: одиночные спайки/потери на промежуточном хопе
  (обычно — рейт-лимит ICMP) игнорируются, флагуются только проблемы,
  сохраняющиеся до конца маршрута. Реализация — в `internal/probe/anomaly.go`,
  UI-агностична.
- ASN-обогащение хопов через бесплатный DNS Team Cymru: вкладка «Маршрут»
  теперь показывает имя автономной системы рядом с хостом
  (`5.180.172.2 · ITNET-AS`, `1.1.1.1 · CLOUDFLARENET`). Приватные IP
  помечаются как «локальная сеть». Резолв идёт в фоне с кэшем; пробы никогда
  не блокируются на DNS. Реализация — `internal/probe/asn.go`. Это также
  фундамент для фазы 4 («Диагноз»).
